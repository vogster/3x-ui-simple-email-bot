
# 📧 3x-ui Simple Email Bot

  

Автоматизированный Email-бот для управления пользователями в панели **3x-ui**. Проект создавался "для себя", чтобы максимально упростить процесс раздачи VPN-доступов друзьям и знакомым без необходимости каждый раз заходить в админку панели.

  

Вся магия происходит через обычную электронную почту: пользователь отправляет кодовое слово - бот мгновенно регистрирует его в 3x-ui и высылает настройки.

  

---

  

## ✨ Основные возможности

  

* 🤖 **Полная автоматизация:** Регистрация новых пользователей в панели 3x-ui без участия администратора.

* 🔑 **Автовыдача конфигов:** Моментальная отправка клиенту готовых ключей / подписок в красивом HTML-письме.

* 📊 **Команда `/status`:** Пользователь может в любой момент узнать состояние своего подключения и остаток трафика.

* ❓ **Команда `/help`:** Быстрое получение справочной информации по настройке клиента.

* 🔔 **Интеграция с Gotify:** Администратор мгновенно получает push-уведомления на телефон о каждой новой регистрации.

* 📢 **Email-рассылки:** Возможность отправить важное объявление всем активным пользователям прямо со своей админской почты.

## 📂 Структура проекта

  

*  `requirements.txt` - список зависимостей (`requests`, `python-dotenv`).

*  `.env.example` - шаблон с переменными окружения (настройки IMAP/SMTP, 3x-ui и Gotify).

*  `config.py` - модуль загрузки и валидации конфигурации.

*  `xui_client.py` - API-клиент для взаимодействия с панелью 3x-ui.

*  `templates.py` - адаптивные HTML-шаблоны для писем (приветствие, трафик, инструкции).

*  `email_bot.py` - основной скрипт бота (опрос ящика, обработка писем и команд).

*  `email-templates` - шаблоны для писем

## 🛠 Настройка и запуск

  

### Шаг 1: Подготовка окружения (`.env`)

  

Создайте файл `.env` на основе шаблона:

  

```bash

cp  .env.example  .env

```

  

Заполните параметры конфигурации:

  

*  `SERVICE_NAME` - название вашего VPN (например, *MyVPN*).

*  `XUI_URL`, `XUI_USER`, `XUI_PASSWORD` - данные для подключения к вашей панели 3x-ui.

*  `XUI_INBOUND_IDS` - ID входящих подключений через запятую (например, `1,2`), куда добавлять новичков.

*  `ADMIN_EMAIL` - ваш Email для получения отчетов и отправки рассылок.

*  `CODEWORD` - секретное слово (например, `BEST-VPN`), которое запускает регистрацию.

*  **Настройки IMAP/SMTP** - данные вашего почтового сервера.

*  **Настройки Gotify** (опционально):

*  `GOTIFY_URL` - адрес вашего сервера уведомлений.

*  `GOTIFY_TOKEN` - токен приложения для отправки пушей.

  

> При использовании Gmail, Yandex или Mail.ru обязательно создайте **Пароль приложения** в настройках безопасности почты. Обычный пароль от ящика работать не будет.

  

### Шаг 2: Установка зависимостей

  

Рекомендуется использовать виртуальное окружение:

  

```bash

python3  -m  venv  venv

source  venv/bin/activate

pip  install  -r  requirements.txt

```

  

### Шаг 3: Запуск

  

Запустите бота в фоновом режиме (через `screen`, `tmux` или как systemd-сервис):

  

```bash
python  email_bot.py
```

## 🚀 Как это работает?

  

### Для пользователя:

  

1.  **Регистрация:** Пользователь пишет письмо на email бота с темой или текстом, содержащим `CODEWORD`. Бот создаёт клиента в 3x-ui и высылает ему настройки.

2.  **Проверка лимитов:** Письмо с текстом `/status` вернет пользователю информацию о потраченном трафике.

3.  **Помощь:** Письмо с текстом `/help` отправит базовые инструкции по подключению.

  

### Для администратора:

  

*  **Уведомления:** При каждой регистрации Gotify пришлет вам пуш: *«Новый пользователь зарегистрирован: user@example.com»*.

*  **Рассылка новостей:** Если вы отправите со своего `ADMIN_EMAIL` письмо на бот-почту с текстом `/broadcast <текст>`, бот автоматически перешлет это сообщение (в красивом оформлении) всем пользователям, которые зарегистрированы в панели.
